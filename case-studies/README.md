# Case studies

Each case study is a self-contained directory:

```
00N-protocol-name/
├── README.md         # the post — intro, data, interpretation, what I did next
├── methodology.md    # how I pulled the data, phase boundaries, confounders
├── phases.json       # machine-readable phase config for analyze-phases.mjs
├── data/
│   └── daily-summary.csv   # anonymized (no user_id, no email)
└── charts/
    └── *.png         # optional visualizations
```

## Format discipline

Every case-study README contains, in this order:

1. **Affiliate disclosure** — any links in the post, with commission disclosure
2. **TL;DR** — one paragraph, the headline finding
3. **What I tried and why** — the protocol, the reasoning
4. **The data** — phase comparison table + daily detail (generated by `analyze-phases.mjs`)
5. **Confounders I can't rule out** — named explicitly
6. **What I'm choosing to do next** — personal decision, not a recommendation
7. **What this does NOT prove** — the most important section
8. **Resources** — affiliate-disclosed and affiliate-free alternatives, side by side
